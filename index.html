Compare Test Data

Test data can be collected from coupons for a build. The user can select tests from across Jobs, materials, build processes, ASTM Test Types; to visualize tabular data, and charted data from the ingested Test data. This can be used to identify best practices when selecting a material/build process for a specific part, and verify part quality.

This can be used to identify best practices when selecting a material/build process for a specific part, and verify part quality. Isolate a printer issue, monitor build quality, or meta data analysis..

STiG Compliant

Utilizes a layered Defense-in-Depth strategy to deter bad actors from gaining system access or intercepting data.

Multiple methods of authentication

Secured authentication including CAC, MFA, and FIDO. Role-based authorization enforced.

Utilizes Encryption for all Data and Connections

Encryption of all Servers and Data at rest. SecureAge is used to encrypt the Application and Database servers. While the database itself has an additional layer of encryption using SQL Server encryption. All connections utilize TLS 1.2.

Hardware encryption used for additional protection

Utilizes StrongKey hardware tokenization of all identifying data (names, phone numbers, emails, etc.). Tokenized data is unreadable if exfiltrated, leaving the remaining data useless.
